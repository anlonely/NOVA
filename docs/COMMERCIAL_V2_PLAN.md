# NOVA INTERP 商业版方案

## 1. 产品定位

NOVA INTERP 不是单纯的“翻译按钮”，而是一个面向 Discord、游戏语音、线上会议、直播和跨语种协作的实时语音路由控制台。

核心价值：

- 双向或多通道独立传译
- 实时双语字幕
- 面向虚拟声卡和复杂音频链路的设备路由
- 低延迟翻译与译后播放
- 针对具体场景的识别偏置与术语纠偏
- 支持声音复刻、预置音色、监控和回放

## 2. 目标场景

- Discord 跨语种语音沟通
- Rust / FPS / 生存类游戏内语音翻译
- KOOK、Discord、游戏语音混合使用
- 主播、会议主持人、双语客服、跨境销售
- 需要“原文 + 译文 + 译音”并行输出的内容团队

## 3. 当前已具备的交付基础

- Qt/Web 版桌面控制台
- Channel A / B / C 独立音频路由
- 自定义深色科技风 UI
- 双栏实时字幕区
- 场景模板、语言切换、延迟抽屉、设置抽屉
- Volcengine AST WebSocket 接入
- 声音复刻 V3 训练与试听链路
- Rust 术语偏置和纠偏入口
- 本地配置持久化
- 会话导出、更新检查钩子
- 原生音频核心 Rust 脚手架

## 4. 商业化还需补强的底座

### 音频底座

- WASAPI 独占/共享模式细化
- 更稳的虚拟声卡与系统默认设备回退逻辑
- AGC、VAD、降噪、噪声门、ducking
- 动态 jitter buffer 与丢包/补偿策略
- 热插拔恢复和长时间运行稳定性

### 翻译与识别

- 更强的领域偏置模板
- 热词、纠错词、术语表批量导入
- 不同通道的独立识别策略
- 更细的延迟指标和链路诊断

### 交付与运维

- 可签名安装包
- 自动更新源与版本清单
- 崩溃恢复与日志打包
- 配置加密与权限隔离
- 商业授权、实例管理、设备绑定

## 5. 延迟目标

建议按模式定义，而不是给一个绝对承诺值：

### Turbo

- 原文首字字幕：200 到 450 ms
- 译文首字字幕：450 到 900 ms
- 首段译音起播：0.9 到 1.8 s

### Balanced

- 原文首字字幕：250 到 550 ms
- 译文首字字幕：650 到 1100 ms
- 首段译音起播：1.3 到 2.4 s

### Studio

- 原文首字字幕：350 到 700 ms
- 译文首字字幕：900 到 1500 ms
- 首段译音起播：2.0 到 4.0 s

说明：

- 声音复刻、24kHz 输出、更重的降噪都会增加端到端延迟
- 游戏语音与平台语音混用时，是否能做到极低延迟取决于系统音频拓扑和虚拟声卡链路

## 6. 商业版建议架构

### 当前版本

- UI：PySide6 + HTML/CSS/JS dashboard
- 控制器：Python runtime orchestration
- 语音翻译：Volcengine AST WebSocket
- 声音复刻：Volcengine Voice Clone V3
- 配置：本地 JSON
- 打包：PyInstaller

### 下一阶段

- UI：保持桌面壳，继续增强 Dashboard
- 音频核心：Rust / WASAPI
- 数据流：Capture -> DSP -> AST -> Subtitle/TTS -> Output Mixer
- Provider 抽象：Volcengine 优先，保留后续替换能力
- 更新系统：GitHub Releases 或私有更新源

## 7. 版本里程碑

### M1 公开可用版

- 稳定的三通道路由
- 可用的桌面 UI
- 配置保存、字幕显示、延迟面板
- Windows 打包与 GitHub 自动产物

### M2 低延迟增强版

- Rust 原生音频核心接入
- 动态补偿 / jitter buffer / 预滚动优化
- 更强的设备恢复与异常诊断

### M3 商业交付版

- 安装包、更新器、签名
- 配置加密、授权体系
- 更稳定的声音复刻链路
- 团队模板和部署文档
